¿Cuáles son las partes de un URL?
Los URL (Uniform Resourse Locator) se refiere a las direcciones de todo lo que se encuentra en internet, es la forma en la que podemos pedir llegar a ciertas direcciones para que recibamos la página que deseamos
Esta se divide en:
Protocol - Aqui se especifica que protocolo tiene, puede ser html para paginas web, ftp para enviar archivos, entre otras
Dominio - Esto es ña dirección principal, es lo que indica a que direcciones IP queremos llegar, se podria decir que es el indice del internet
Path - Es la subdireccion de la pagina, se usa para llegar a las diferentes subpaginas del dominio principal
Parametros - Es información extra que ayuda a que la página funcione correctamente

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
Todos los requests de HTML tienen que tener un URL y la respuesta tiene que tener un metodo
Get - Se usa para obtenet una página según la dirección URL
Head - Obtiene solamente el titulo de la página
Post - Sube información a cierta página
Put - Actualiza un elemento de la página
Patch - Solamente crea cambios parciales en algun elemento del programa
Delete - Elimina elementos de un programa

¿Cuál es el ciclo de desarrollo de sistemas de información?
El ciclo de vida de los sistemas es la vida completa de un sistema, este es desde que se descubre la idea o el problema que se quiere solucionar, de aquí también abarca a todo el proceso de desarrollo, de publicación y sigue abarcando todo la vida de uso, y termina hasta que se descontinúa
Mientras que el ciclo de vida de desarrollo es solamente la parte en que el programa se crea, este igualmente abarca multiples etapas pero solamente va desde la concepción de los requerimientos hasta que el programa deja de estar en desarrollo, osea, cuando se publica
