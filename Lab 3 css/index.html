Questions:

  • Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?
  • Es util pero no se debe de abusar, sería útil para establecer partes de diseño que es muy importante que se vean siempre igual, por más que agreges elementos o escribas cosas de mayor prioridad.
    Otro elemento que se ve útil es para las preferencias del usuario, las cuales siempre deben de tener mas prioridad que las de un programador.
  • Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?
  • 1, debes asegurarte que la imágen esté disponible
    2, probar que la imágen se vea bien de fondo y no haga el texto difícil de leer
  • Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?
  • Es más recomendable usar una unidad de medida que sea relativo a la pantalla pues asi nos evitamos problemas al usar un dispositivo mas pequeño o grande, y que el contenido salga similar en todas las pantallas. Fuera de eso, el % o la unidad em suelen ser las recomendadas
  • ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?
  • Porque cuando cargar un archivo sin comprimir también estás enviando los tabuladores, saltos de línea, etc, lo cual es información que estás enviando la cual es inútil o basura para el funcionamiento de la página, mientras que CSS puede funcionar perfectamente sin esos tabuladores o espacios de mas
